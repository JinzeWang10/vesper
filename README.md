# 日省 · Vesper

> 一个长在你编码 agent 里的极简反思 skill ——
> **每天收工时，只问你一个问题，然后安静退场。**

---

## 这是什么

当所有 AI 工具都在帮你「做更多、追更快」的时候，`Vesper` 反着来：
它在你一天 vibe coding 结束、正要合上电脑的那一刻，**安静地问你一个关于你自己的问题**，
收下你的回答，**不评判、不建议、不啰嗦**，然后退场。

问题不是随机的。它**长在你过去的回答之上**——像一面镜子，
用得越久，越懂你，问得越深；偶尔把旧答案照回来给你看。

名字取自《论语》曾子「吾日三省吾身」，
对上苏格拉底「未经省察的人生不值得过」。
东西方都在说同一件事：**每日自省。**

## 为什么是这个形态

- **反 FOMO**：你正向外狂奔，它把你拉回来一秒，问你一句关于你自己的事。
- **极简 / 沉浸 / 放松**：每天一个问题，只问一次。没有打卡天数、没有分数、没有社交、没有多巴胺。
- **Local-first，说诚实的那种**：你的回答全部存在你自己的设备上（`~/.vesper/`）。
  Vesper 自身**零服务器、零上传、零遥测**——没有任何数据因为装了它而多去一个地方。
  诚实说明：生成问题时，你的回答会作为上下文经过**你本来就在用的那个编码 agent 的模型 API**（和你的代码走同一条通道，不新增任何第三方），也会像你对 agent 说的任何话一样留在它的会话记录里。用本地模型跑 agent，则全程真·本地。
- **长在你已经待的地方**：作为编码 agent 的 skill，不用再装一个 App，不用打流量仗。

## 核心机制

```
每天收工（默认傍晚 session 结束，可自定义）
   ↓ 一个由你历史回答生成的问题（苏格拉底式，向内）
   ↓ 你答
   ↓ 它安静收下：「记住了，明天见」（当场不回应）
   ↓ 存到本地
   ↓ 周期性（每周/每月）「镜子时刻」：回看你的回答，把模式照给你看（带出处）
```

# 日省 · Vesper — v0 设计

- 日期：2026-06-17
- 状态：设计已收敛，待写实现计划
- 作者：王金泽

---

## 0. 一句话

一个长在编码 agent 里的极简反思 skill：每天收工时只问用户一个（基于其历史生成的）问题，收下回答、当场不回应，数据全部留在本地；周期性回看，把模式照回给用户。

## 1. 为什么做这个（thesis）

- AI 飞速迭代，人人 FOMO；但「追得再快也赶不上技术迭代」。
- 当所有工具都在帮人「向外做更多」，本产品反着来：用 AI 帮人**向内看、回归本质、关注自己**。
- 这是作者一以贯之的母题（knowyourself 心理量表、八戒戒断、以及作者自己在用的本地 AI 教练）。
- 逆向定位：别人做「AI 帮你向外扩张」，我做「AI 帮你向内安住」——在 FOMO 的世界里稀缺且戳人。

## 2. 目标与非目标

**目标**
- 首要：成为强 FDE 候选人的**开源作品集 + 可 demo 的工程叙事**（agent + 个人数据 RAG + eval + 可暴露成 MCP server）。
- 其次：一个作者自己真心想用、也愿意给同类开发者用的工具。

**非目标（v0 明确不追）**
- 不追消费级流量、不打分销战。受众是开发者/极客，分销内建在「他们已经待的编码 agent 里」。
- 不需要「真实用户规模」来证明价值——价值 = 漂亮的 repo + 能打开的 demo + 想清楚的设计。

## 3. 形态决策

**主体：编码 agent 的 Skill + hook。**
- repo 内含技能定义 + 本地记忆文件（参考作者在用的 resume-coach 的 `sessions/` + `state/` 结构）。
- 用 hook 实现「每天收工自动问一次」。
- 加分件（后续）：把记忆层暴露成 **MCP server**，跨客户端复用，同时正面补作者简历的 MCP 缺口。

**为什么不是 App / Web / CLi 独立形态**：那些都要解决「让人来用」的分销问题；长在编码 agent 里则把分销问题直接消解，且「在 FOMO 编码中插入一次向内的打断」本身就是主题最锋利的表达。

## 4. 核心交互（铁律，不可破）

1. **每天一个问题，只问一次。** 没法暴食——反 FOMO 写进交互。绝不提供「再来一题」。
2. **答完当场不回应。** 只温柔收下（「记住了，明天见」）。无多巴胺、不让人表演给 AI 看。
3. **极简 / 沉浸 / 放松。** 砍掉一切制造压力的东西：**无连续打卡天数、无分数、无社交、无多余推送**。一天一个温柔提醒足矣。可跳过、绝不追问。
4. **Local-first。** 回答是最私密内心数据，**全部存本地，永不上传**（已在 `.gitignore` 排除 `data/`）。

## 5. 智能在哪（产品内核 + FDE 故事）

**① 问题长在历史之上（不是随机/固定）**
- 由 agent 读用户过去的回答来生成今天的问题：顺着上次往深问、偶尔把旧答案照回来（**带出处**：「三周前你也说过一模一样的话」）。
- 用得越久越懂你 → 真·数据飞轮，且攒的是用户的自我认知。
- 技术映射：个人回答库 RAG + 记忆 + 带出处引用。

**② 两种节奏分开**
- **每日**：问 → 答 → 安静收下（保「放松沉浸」）。
- **周期性（每周/每月）「镜子时刻」**：回看一段时间的回答，把模式照给用户看（「这个月你提到『累』7 次，多在周日晚上」）。价值感与惊喜攒在这里。

**③ 长期礼物**：攒满一年 → 「你的一年，52 个问题与回答」回顾，让积累有重量。

## 6. 触发机制（已精修）

**原则**：早上是「启动模式」，静不下来反思；反思的本质是回看，必须在事情发生**之后**。所以默认**收工**问。

**核心判断：PULL 管够，PUSH 克制。**
- **PULL（人主动来）**：零打扰成本，多多益善——不违反铁律，是人自己要的。
- **PUSH（系统去推）**：每加一个都在赌「每天只问一次 / 不打扰」，越多越吵。
- 目标不是「加触发」，而是 PULL 丰富、PUSH 只留一个。（这是对早期「自动为主、手动兜底」的反转。）

**触发矩阵**

PULL（慷慨给）：
1. `/收工` 显式命令。
2. **自然语言收工意图**——人收工本来就会说话：「今天到这吧」「下班了」「不写了」「累了」「good night」「wrapping up」…… 在 skill description 里登记触发短语（参考 resume-coach），模型识别后温柔接问。入口从「一个命令」变成「任何一句收工的话」，零打断、跨 agent 通用。
3. **随时主动领题**——「默认收工才问」只约束 PUSH；人想中午/做完 feature 时反思，一句话即可 pull 出今天的问题。
4. `/镜子`（或 `/回看`）主动看周期回看。

PUSH（只留一个）：
1. 傍晚 Stop-hook 一句**提醒**（非问题本身）：过了傍晚阈值、当天没提醒过 → 轻附「天色晚了，今天想日省一下吗？`/收工` 领今天的问题」。一天一句，对上「一天一个温柔提醒足矣」。

**回看/年度礼物搭车**：答完今天问题、温柔收下那刻，若攒够一周/月/年，顺口附一句「这周镜子攒好了，想看吗？」，不单独 PUSH。

**关键技术约束**：
- Skill 由模型经 Skill 工具调用或用户输入 `/命令`；Hook 是确定性 shell 脚本，**没法直接替模型调 Skill、替用户输入 `/`**。
- ❌「SessionEnd 时问」不成立：会话正在关闭、无下一轮给模型说话，输出只用于清理。
- ❌ 静默检测（B）：需后台守候、hook 不擅长、「静默 ≠ 收工」太脆。
- 唯一可行自动桥接是 `Stop` hook（会话还活着、一轮刚结束）。但 Stop ≠ 收工，是「过阈值后第一次答完话那刻」；若此时抛**完整问题**等于打断投入（「误触发无害」对完整问题不成立），**所以自动只输出一句提醒**。
- 死循环坑：Stop hook 注入会再触发自己，hook 输入有 `stop_hook_active` 字段，为 true 时脚本直接放行。

**数据与每日闸必须用户级、非 repo 级**：一天在多 repo 写代码，标记/回答库若存每个 repo 的 `data/` 会刷屏。放用户全局位置（如 `~/.vesper/`），repo 里只放 skill 代码。亦印证 MCP 加分项（记忆本就用户级、跨项目）。

**可移植性**：`Stop` hook 是 Claude Code 概念，别的 agent 不一定有。自动提醒只在 Claude Code 跑；`/收工` 与自然语言意图才跨 agent 通用——又一个「手动/自然语言为主」的理由，README 讲清。

**自定义空间**（用户可调）：傍晚阈值时间点、自动提醒开/关、提醒口吻等。

**v2（YAGNI，v0 不做）**：让 skill 观察用户平时几点收工，自动学会在其节奏点提醒；「提交/ship 成功后触发」（PostToolUse）。

**待实现时核实**：`Stop` hook 在各 agent 里的精确语义、注入提醒的具体 JSON 字段、关窗算不算 session 结束。

## 7. 技术骨架（待写实现计划时细化）

- 本地存储：用户回答 + 元数据（时间戳等）落本地文件，`data/` 已 gitignore。
- 问题生成：读历史 → RAG/检索相关旧答案 → 生成苏格拉底式新问（带出处）。
- 触发：hook（傍晚 + session 结束 + 每日一次闸）+ 手动命令。
- 周期回看：扫一段窗口的回答 → 归纳模式 → 带出处呈现。
- 加分：MCP server 暴露记忆/反思能力。
- 模型与运行位置（本地模型 vs 云端、隐私边界）：**实现阶段再定**（涉及 LLM 选型时核对 claude-api 参考）。

## 8. 语言策略（单一通用 skill，不做两套）

**结论：做一套通用 skill，语言是运行时细节，不是两个版本。**

- **拆解后没有一样需要 fork 代码**：触发机制 / 本地存储 / RAG / schema 全部语言无关；真正随语言变的（问题、「收下」语、镜子归纳、触发短语）要么是 **LLM 运行时生成**，要么是一个文案变量。做两套 = 维护翻倍、数据分裂、demo 碎片化，为不存在的差异买单，违背极简内核。
- **语言跟随用户数据自动切**：用户用什么语言答，历史回答库就是什么语言，RAG 喂回去模型自然用同语言问。语言「长在数据里」，与「用得越久越懂你」是同一机制，用户无需选择。
- **唯一分两份的是数据、不是代码**：中文 / 英文各一份**种子问题集**（仅冷启动 1~7 天用到），挂在同一套 skill 上。
- **分层**：skill 内核单一通用；种子集中英两份数据文件；对外门面（README / demo / MCP 描述）面向 FDE 作品集受众，英文优先 / 双语。
- 冷启动语言判定：读 agent locale，或首题用双语种子，之后跟随用户回答语言。
